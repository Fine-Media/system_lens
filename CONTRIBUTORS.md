## Community acknowledgments

We also welcome contributions that are not only code:

- Issue reports and reproduction steps
- Documentation and tutorials
- Reviews and design feedback
- Translations and accessibility improvements

If you would like to be listed here for **non-code** contributions, open a pull request that adds a short line under **Community acknowledgments** or ask a maintainer in an issue.

## How to become a contributor

1. Read [CONTRIBUTING.md](CONTRIBUTING.md) and the [Code of Conduct](CODE_OF_CONDUCT.md).
2. Pick an issue labeled `good first issue` or `help wanted` from [GitHub Issues](https://github.com/OWNER/REPO/issues) (update the link after you publish the repo).
3. Open a pull request with a clear description and a passing `npm run build`.

If you are new to open source, say so in the PR — reviewers will help you through the process.
